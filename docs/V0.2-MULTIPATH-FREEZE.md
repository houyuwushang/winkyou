# v0.2 Multipath / Bootstrap Freeze Gate

This document defines the v0.2 freeze boundary for the protected-direct multipath and intermittent bootstrap work. It is a gate, not a proposal for expanding the data plane.

Active architecture remains:

```text
WinkYou = connectivity solver + secure WireGuard data plane
```

The authority for session, solver, strategy, transport, and tunnel boundaries remains [`CONNECTIVITY-SOLVER-BASELINE.md`](./CONNECTIVITY-SOLVER-BASELINE.md).

## Freeze Intent

v0.2 should prove that a selected primary path can be quality-driven while an established direct/P2P path is preserved as protected standby. It should also prove that coordinator or broker loss is not confused with data-plane failure.

The release should make the following visible to operators:

- which path is primary;
- whether a protected direct standby exists;
- which child path is active after failover;
- whether in-band heartbeat/path_health is keeping peer state alive while coordinator state is degraded.

## Completion Conditions

Before v0.2 can freeze, all of the following must be true:

- `pkg/transport/multipath` unit tests pass, including primary write, standby fallback, read fan-in, shadow write, health control, and close behavior.
- Fake session/client tests pass for `relay_only` primary plus direct protected standby.
- `connectivity.multipath.enabled` is configurable and defaults to the protected-direct policy in `auto` mode: `enabled=true`, `max_paths=2`, and `shadow_write=true`. Relay-only policy remains single-path.
- `wink peers` and `wink doctor` display primary path, protected direct path, standby paths, active path, and failover state.
- In-band heartbeat/path_health has a minimal runtime loop and is reflected in runtime JSON and peer state.
- Intermittent bootstrap broker work is limited to message models and the memory simulator; it is not wired into default production behavior.
- The real-device test report template is updated. For this phase, the JSON report emitted by [`scripts/verify-multipath-failover.py`](../scripts/verify-multipath-failover.py) is the expected report shape.

## Required Verification

Run the standard regression gate:

```bash
git diff --check
go fmt ./...
go vet ./...
go test ./... -count=1
```

For core client/session/transport changes, also run:

```bash
go test -race ./pkg/session ./pkg/client ./pkg/solver/... ./pkg/transport/... ./pkg/tunnel/... -count=1
```

On Windows, race tests require cgo and a C compiler such as GCC.

Run the multipath failover verifier in dry-run mode before using any destructive action:

```bash
python scripts/verify-multipath-failover.py --help
python scripts/verify-multipath-failover.py --peer <peer-name-or-id>
```

Only pass `--stop-coordinator`, `--stop-relay`, or `--pause-primary-host` with the explicit confirmation flag in a controlled lab.

## Real-Device Evidence

The minimal real-device topology remains:

```text
A = local client
B = intermittent helper / coordinator / relay host depending on the test
C = remote gateway client
```

Do not commit host passwords, private keys, local SSH aliases, or site-specific addressing into the repository.

The report should show:

- baseline `wink peers --json`;
- baseline `wink ping <peer>`;
- `multipath_enabled=true`;
- `protected_direct_path_id` is non-empty when the test claims protected standby coverage;
- fault action used, or `dry_run=true`;
- multiple post-fault ping attempts;
- final peer state including active path, protected direct path, and last failover time.

## Non-Goals

v0.2 does not include:

- default peer relay;
- arbitrary nodes forwarding other peers' user traffic;
- QUIC, HTTP CONNECT, or WebSocket strategies;
- public route table updates;
- self-developed data plane;
- automatic router control;
- unauthorized route injection;
- mobile or GUI work.

## Bootstrap Boundary

The intermittent bootstrap broker is allowed to model store-and-forward rendezvous for A/B/C scenarios:

- B can cache A's descriptor while connected to A.
- B can later deliver A's descriptor to C and collect C's descriptor.
- B can later return C's descriptor to A.
- A and C should establish their own direct or multipath data plane.

B must not become the default A-C user-data relay. Production integration remains outside this freeze unless a later gate explicitly authorizes it.

## Exit Criteria

v0.2 is ready to freeze when the completion conditions pass, the real-device report demonstrates protected direct standby behavior or clearly documents why direct standby is unavailable, and no open issue requires enabling peer relay, custom data-plane work, or public routing changes to satisfy the phase objective.
